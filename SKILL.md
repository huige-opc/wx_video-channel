# wx-channel-cleaner —— 视频号逐字稿 AI 清洗

## 何时使用

用户说这些话时触发：
- "清洗"、"清洗视频稿"、"清洗视频号"、"整理逐字稿"
- "清洗 XX"（XX 是 downloads/ 下的实际文件名或其中一段）
- "把新下载的视频稿清洗一下"

## 工作目录（唯一）

- **输入目录**：`d:\AI_bian_cheng\trae\tools\wx_video-channel\downloads\`
- **原始稿命名**：`xxx.md`（由 [transcribe.py](file:///d:/AI_bian_cheng/trae/tools/wx_video-channel/transcribe.py) 生成，同名 `.mp4` `.mp3` 相邻）
- **清洗稿命名**：`xxx_清洗版.md`

## 执行步骤

### 1. 扫描待清洗文件

用 `Glob` 找出所有满足条件的原始稿：

```
downloads/**/*.md
```

**过滤规则**：
- 排除文件名含 `_清洗版` 的（已清洗过的跳过）
- 排除对应位置已有同名 `_清洗版.md` 的（防重复清洗）

如果没有待清洗文件，直接回复："没有需要清洗的稿子"。

### 2. 对每一份原始稿

- 用 `Read` 读取全文
- 分析上下文，做清洗（详见"清洗规则"）
- 用 `Write` 写入同目录下 `xxx_清洗版.md`
- 用 `DeleteFile` 删除原始 `xxx.md`

### 3. 报告结果

一段话总结：清洗了几个、每个字数、保留了哪些原意不确定的地方。

## 清洗规则

### 🔴 必须做的修正

1. **错字修正**（基于上下文推断）
   - `哎呀` / `ar` / `ai` / `Ai` → **AI**（几乎肯定是 AI）
   - `扣扣扣扣` / 拟声重复 → 保留一次或删除
   - 字母缩写乱码（`paypal` `asmr` `ootis` `oot麦` 等）→ 结合上下文推断，实在不确定则**保留原文并加脚注 `[原音频含糊]`**
   - `9` → `就` / `就有` （数字被误识别成实词）
   - 数字读法误识（`900` → `就有多`、`八弟` → 拟声词或产品名）

2. **补/删标点**
   - 长句子拆成短句
   - 排比结构用列表
   - 反问句加"？"

3. **段落结构化**
   - 有编号或"第一/第二/第三"的地方，用 `### 一、` `### 二、` `### 三、`
   - 列举内容用有序列表 `1. 2. 3.` 或无序列表 `- - -`
   - 核心观点用 `**加粗**`
   - 金句放 `> 引用块`

4. **保留元信息头**（原稿最上面 `# 标题 - 作者 - 来源 - 视频文件` 那部分**必须保留**）

### 🟢 必须保留

- **所有事实内容**（观点、案例、数据）
- **说话人的原意**（口语化风格可以调整为半书面，但核心表达要在）
- **不确定的专有名词**（宁可保留原文加脚注，不要瞎猜）

### 🔴 严禁做的

- ❌ 添加原文里没有的内容
- ❌ 删除原文里的观点/事实
- ❌ 用自己的话完全改写
- ❌ 添加评论或点评

## 输出模板

```markdown
# {原标题}

- **作者**：{作者}
- **来源**：微信视频号
- **视频文件**：`{文件名}.mp4`

---

## 逐字稿（AI 清洗版）

{开头/引言段落}

### 一、{小节 1 标题}

{正文...}

### 二、{小节 2 标题}

{正文...}

---

> **核心观点**：{金句，如果原稿明显有的话}

---

*说明：本文由百度短语音识别 API 转录后，基于上下文进行了 AI 清洗和结构化。{如有不确定的专有名词，在此列出}。*
```

## 参考样本

参照上文「输出模板」的结构和「清洗规则」的要求做后续清洗：保留元信息头、修正错字、补标点、用 `### 一、二、三` 分小节、核心观点加粗、金句用引用块，深度和口吻贴近原稿的口语风格。

## 完成后的目录状态

每个作者目录最终**只留 3 个文件**：

```
downloads/{作者}/
├── xxx.mp4         视频
├── xxx.mp3         音频
└── xxx_清洗版.md   AI 清洗后的逐字稿
```
